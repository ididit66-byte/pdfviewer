# 변경 이력 (Changelog)

이 프로젝트의 주요 변경사항을 기록합니다.
형식은 [Keep a Changelog](https://keepachangelog.com/ko/1.1.0/)를 따르며,
버전은 서비스워커 캐시 버전(`sw.js`의 `CACHE`)과 일치시킵니다.

## [v4] - 2026-07-01 · `1b5e4db`
### 추가 (Added)
- **최근 파일 기록** — 연 PDF의 내용을 기기 내부(IndexedDB)에 저장해, 시작 화면 목록에서 탭하면 실제로 다시 열립니다. (경로로 재접근이 불가능한 웹앱 제약을 우회)
  - 목록에 파일 이름·크기·마지막으로 연 시간·이어보기 지점 표시
  - 항목별 삭제(✕) 및 전체 지우기, 최대 12개 보관(초과분 자동 정리)
- **이어보기** — 마지막으로 본 페이지를 기억해 다시 열 때 그 페이지부터 표시.
- 툴바에 **홈(🏠) 버튼** — 열린 문서 위로 최근 목록을 다시 표시.

### 변경 (Changed)
- 80MB 초과 파일은 목록엔 남기되 "재열기 불가"로 표시 (저장 용량/쿼터 보호).

### 수정 (Fixed)
- 시작 화면(welcome) 오버레이가 투명이라, 열린 문서 위에 띄웠을 때 뒤쪽 PDF로 클릭이 새던 문제를 불투명 처리로 수정.

## [v3] - 2026-07-01 · `5a27686`
### 수정 (Fixed)
- **확대 상태에서 좌우로 밀면 이전 페이지로 넘어가던 문제** 수정.
  - 넘김/패닝 판단을 제스처 '끝'이 아니라 '시작' 시점의 확대 상태로 결정 → 확대 중 시작한 드래그는 항상 패닝.
  - 제스처 도중 손가락이 추가되면(핀치 전환) 넘김 후보에서 즉시 제외.
  - 핀치 종료 시 스와이프 핸들러가 먼저 실행되며 옛 배율/좌표로 페이지를 넘기던 타이밍 경쟁(race) 제거.

## [v2] - 2026-07-01 · `416aea0`
### 변경 (Changed)
- **핀치 확대를 부드럽게 개선** — 확대 중 CSS 변형으로 실시간 미리보기(손가락을 즉시 따라감), 전체화면 로딩 스피너 제거, 재렌더는 화면 밖에서 처리해 깜빡임 없음.
- 확대/축소 시 화면 중심을 유지하도록 스크롤 보정.
- 확대 상태에서는 가로 스와이프를 페이지 넘김이 아닌 패닝으로 처리.
- 연속 확대 요청을 직렬화해 재렌더 겹침 방지.

### 수정 (Fixed)
- **확대 후 좌우/상하 이동이 안 되던 문제** 수정 — `.pages`를 `width:max-content`로 바꿔 확대 시 좌측 오버플로가 잘려 스크롤로 도달하지 못하던 flexbox 버그 해결, `viewer`에 `touch-action:pan-x pan-y`를 적용해 한 손가락 상하좌우 이동 허용.

## [v1] - 2026-07-01 · `bba4402`
### 추가 (Added)
- 최초 버전: PDF.js 기반 스마트폰용 PDF 뷰어 (PWA).
  - 파일 열기(파일 선택 + 드래그 앤 드롭)
  - 연속 세로 스크롤 보기, 페이지 번호 직접 이동, 좌우 스와이프 넘김
  - 확대/축소(버튼·핀치), 화면 너비 자동 맞춤
  - PDF 내장 목차(북마크) 이동
  - 문서 내 텍스트 검색(하이라이트·이전/다음)
  - PWA: 홈 화면 설치 및 서비스워커 오프라인 캐시
